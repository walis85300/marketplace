1. Run `git status` to see current changes
2. Show me a summary of what changed
3. Generate a conventional commit message based on the changes
4. Ask me to confirm or edit the message before committing
5. Do NOT push — only commit locally

If $ARGUMENTS is provided, use it as a hint for the commit scope or type.
